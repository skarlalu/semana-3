En este repositorio podemos encontrar el taller el cual está dividido en dos partes: programación tradicional y programación poo
El archivo de programación tradicional resuelve el problema utilizando funciones independientes y captura de datos por teclado.
El archivo de programación poo resuelve el problema aplicando abstracción, clases, atributos, métodos y modularidad en archivos separados 
En la programación tradicional, el código se enfoca en los pasos o procedimientos secuenciales (funciones y variables sueltas). En cambio, la POO organiza el código alrededor de objetos de la vida real, agrupando de forma lógica tanto los datos como sus comportamientos en una sola entidad.
Separar el programa POO en archivos (mascota.py y main.py) demuestra cómo este paradigma facilita el mantenimiento. Es mucho más sencillo extender las capacidades del sistema en un entorno POO sin alterar la lógica principal del programa.
